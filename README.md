## 未上传的文件

以下文件未上传到仓库，因为不适合放Git：

### 1. 数据文件 (.csv)
- `data/红酒品质分类.csv`
- `data/红酒品质分类_train.csv`
- `data/红酒品质分类_test.csv`
- `data/wine.csv`
- `data/titanic_train.csv`

**原因：**
- CSV是二进制或文本格式的原始数据
- Git对大文件处理效率低
- 占用仓库体积

**替代方案：**
- 数据放在本地，通过 `.gitignore` 忽略
- 或使用云存储（如百度网盘）单独备份
- 或使用数据版本管理工具（如DVC）

### 2. 模型文件 (.pkl)
- `model/红酒品质分类.pkl`

**原因：**
- 二进制序列化文件，无法对比修改
- 占用仓库体积

**替代方案：**
- 模型放在本地，通过 `.gitignore` 忽略
- 或使用模型托管平台（如Hugging Face）

---

## .gitignore 配置建议

```gitignore
# 数据文件
*.csv

# 模型文件
*.pkl
*.pickle
*.joblib

# 其他
__pycache__/
*.pyc
```

---

## 正确的工作流程

```
1. 数据准备（本地）
   └── 清洗好的数据保存到本地

2. 模型训练（本地）
   └── 用代码读取数据，训练模型
   └── 保存模型到本地

3. 代码同步（GitHub）
   └── 只上传 .py 代码文件
   └── 数据和模型放本地，不上传
```

---

## 总结

| 文件类型 | 处理方式 |
|----------|----------|
| .py 代码 | 上传到GitHub |
| .csv 数据 | 本地保存，不上传 |
| .pkl 模型 | 本地保存，不上传 |
