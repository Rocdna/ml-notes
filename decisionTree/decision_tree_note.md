# 决策树学习笔记

---

## 1. 什么是决策树？

**决策树**是一种树形结构的机器学习算法，通过对数据特征的层层划分来进行预测。

### 直观理解

想象你在玩"20个问题"游戏：
- 对方想一个东西，你提问来猜
- "是动物吗？" → "是" → "会飞吗？" → ...
- 每个问题就是一个**节点**，回答"是"走左分支，"否"走右分支
- 最终得到答案（叶子节点）

决策树就是这个原理，不断问"这个问题"来缩小范围，直到得出结论。

## 2. 分类树：预测类别

### 2.1 ID3 决策树

**核心思想**：用**信息增益**最大的特征来分裂。

#### 熵 (Entropy)

**定义**：衡量数据的混乱程度、不确定性

```
熵越大 → 数据越混乱 → 不确定性高
熵越小 → 数据越纯 → 不确定性低
```

**公式**：
$$H(X) = -\sum_{i} P(x_i) \log_2 P(x_i)$$

**例子**：

| 情况 | 熵值 |
|------|------|
| 所有样本同类 | 0（最纯） |
| 各类各占一半 | 1（最混乱） |
| 三个类别各1/3 | 1.58 |

#### 信息增益 (Information Gain)

**定义**：划分前后熵的变化，**增益越大，划分越有效**

$$g(D, A) = H(D) - \sum_{t} P(t) \cdot H(D_t)$$

**简单理解**：
```
信息增益 = 划分前熵 - 划分后熵
增益大 = 划分后数据更纯了
```

#### ID3 的问题

> **偏向于选择取值多的特征**（如ID号、日期），容易过拟合

---

### 2.2 C4.5 决策树

**核心思想**：用**信息增益率**代替信息增益，惩罚取值多的特征。

**公式**：
$$信息增益率 = \frac{信息增益}{特征熵}$$

**解释**：
- 特征取值越多 → 特征熵越大 → 惩罚越重 → 增益率变小

---

### 2.3 CART 决策树（分类）

**核心思想**：用**基尼系数**来选择分裂特征。

#### 基尼系数 (Gini)

**定义**：衡量数据集的纯度，基尼系数越小，纯度越高

**公式**：
$$Gini(D) = 1 - \sum_{t} P(t)^2$$

**与熵的关系**：
- 两者作用类似，都是衡量混乱程度
- 熵计算稍复杂（含对数），基尼系数更简单
- CART 默认使用基尼系数（计算效率高）

## 3. 回归树：预测连续值

### 3.1 与分类树的区别

| | 分类树 | 回归树 |
|--|--------|--------|
| 预测目标 | 类别（离散） | 连续值 |
| 叶子节点 | 类别标签 | 数值（均值） |
| 分裂准则 | 熵、基尼系数 | 方差、MSE |

### 3.2 分裂准则：为什么用方差/MSE？

**核心原因**：回归目标是让预测误差最小，而 MSE 正好是误差的度量。

**公式**：
$$MSE = \frac{1}{n} \sum_{i=1}^{n} (y_i - \bar{y})^2$$

#### 具体流程

1. **计算初始方差**：所有样本目标值的方差

2. **遍历所有划分点**：
   - 对连续特征排序，相邻值中间就是划分点
   - 计算每个划分点的加权 MSE

3. **选择最佳划分**：
   ```
   划分点1: 85㎡ → 左:A | 右:B,C,D,E → MSE=280
   划分点2: 95㎡ → 左:A,B | 右:C,D,E → MSE=231.67
   划分点3: 105㎡ → 左:A,B,C | 右:D,E → MSE=133.33
   划分点4: 115㎡ → 左:A,B,C,D | 右:E → MSE=113.33 ✓
   ```

4. **选择 MSE 最小的划分点**（方差减少最多）

### 3.3 预测值

**叶子节点预测值 = 该叶子所有样本目标值的均值**

因为数学上可以证明：使得 MSE 最小的预测值就是均值。

## 4. 决策树剪枝

### 4.1 为什么要剪枝？

**问题**：决策树如果不限制，会一直分裂下去，容易**过拟合**

**过拟合**：像背答案的学生，练习题都会，但考试新题不会

### 4.2 预剪枝 (Pre-pruning)

**时机**：树生长过程中

**方法**：
- 限制最大深度
- 限制节点最少样本数
- 信息增益阈值

**优点**：训练快，省资源
**缺点**：可能过早停止，欠拟合

### 4.3 后剪枝 (Post-pruning)

**时机**：树完全长成后

**方法**：从叶子往上，逐个尝试剪掉子树

**常用算法**：CCP (Cost Complexity Pruning)
$$Score = MSE + \alpha \times 叶子节点数$$

- α 大 → 偏好小树（多剪）
- α 小 → 偏好大树（少剪）

**优点**：泛化能力更好
**缺点**：训练时间开销大

| 算法 | 分裂准则 | 类型 |
|------|----------|------|
| ID3 | 信息增益 | 分类 |
| C4.5 | 信息增益率 | 分类 |
| CART | 基尼系数 | 分类 |
| CART回归 | MSE/方差 | 回归 |
